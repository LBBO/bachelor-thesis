\myabstract{
Modern web development relies highly on third-party dependencies, such as front-end frameworks or component libraries. Enabling all these dependencies to work together is often a tedious and repetitive task. Therefore, this thesis aims to automize such initial creation of web projects.

After comparing current approaches to this problem, a new command-line interface (CLI) will be created to guide the user through a small questionnaire, prompting them for their choice of tools and configurations. The questionnaire will both prevent the user from choosing discouraged configurations and provide them with additional helpful information about each question in order to lower the bar of entry for new developers. The CLI will then create a new project according to the chosen specifications.

Using aspects of test-driven development, functional programming and graph theory, the implementation will focus on creating an extension-based framework for asking questions and installing dependencies. After also developing a small set of extensions to demonstrate the framework's capabilities, a series of end-to-end tests will be created and executed in order to judge the CLI's success. In the end, total set of 56 different configurations will be successfully installable via the created CLI.
}
